# ssc-500-resource

## 项目简介

ssc-500-resource 是一个用于归档和发布多个独立 HTML 页面的资源仓库。本仓库不针对任何特定网站或域名，仅作为静态 HTML 文件的集中存放与版本管理空间。

仓库中的每一个 HTML 文件均为独立完整的页面，不依赖外部框架或数据库，可直接在浏览器中打开查看。

## 目录说明

```
ssc-500-resource/
├── pages/          # 存放所有独立 HTML 页面文件
├── assets/         # 页面所需的公共资源（图片、样式、脚本等）
└── README.md       # 本说明文件
```

- `pages/` 目录下每个文件或子目录对应一个独立页面，命名清晰，便于识别。
- `assets/` 目录存放多个页面可能共用的资源文件，避免重复上传。

## 页面归档说明

- 所有页面均以 `.html` 后缀保存，编码统一为 UTF-8。
- 每个页面均可在本地直接打开运行，无跨域或远程资源依赖。
- 页面内容不包含任何指向具体域名或商业网站的链接，仅作为纯展示用途。
- 归档遵循“一页一文件”原则，不进行页面合并或嵌套。

## 维护说明

- 欢迎提交 Pull Request 添加新的独立 HTML 页面，请确保页面文件放置在 `pages/` 下，并保持文件结构清晰。
- 请勿在页面中包含外部跟踪脚本、广告代码或任何形式的用户数据收集功能。
- 如需更新已有页面，建议直接替换文件并注明变更内容。
- 仓库维护者会定期检查页面可用性，但不对页面内容本身承担审核义务。

## 使用建议

- 可直接将 `pages/` 下的 HTML 文件部署至任意静态托管服务。
- 也可通过 GitHub Pages 功能启用本仓库的页面展示。
- 如需引用公共资源，请使用相对路径（如 `../assets/`）。
